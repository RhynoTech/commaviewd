# Changelog

## [0.0.55-alpha] - 2026-07-11

### What changed

- Added the restored stable runtime release for comma 3/3X and comma four device testing.
- Added platform-aware onroad UI export hook installation so comma four/MICI uses the MICI UI hook target and comma 3X/TIZI plus legacy comma 3/TICI use the TIZI hook target family.
- Switched the canonical comma-device companion path from `comma4/` to `comma/`.
- Kept a backwards-compatible `comma4/install.sh` shim so older Android app builds can still fetch the installer and be forwarded to the new `comma/install.sh` path.
- Kept this release on the stable TCP-era app/runtime interface; the UDP transport work remains parked on the UDP feature branch.

### Compatibility

- Compatible Android app: `0.0.157-alpha` / tag `app-v0.0.157-alpha`.
- Supported runtime install platforms: `mici`, `tizi`, and legacy alias `tici`.
- Supported upstreams: official `commaai/openpilot` and known SunnyPilot remotes.
- Current comma 3X support should use the TIZI path; legacy comma 3/TICI is kept as a backwards-compatible alias for the same hook family.

### Testing focus

- Fresh install and repair on one comma four/MICI device and one comma 3X/TIZI or legacy comma 3/TICI device.
- Confirm runtime patch status reports the expected `uiPlatform`, raw `deviceModel`, and selected augmented-road hook target.
- Confirm pairing, live view, camera switching, overlays, and runtime repair actions from the Android app.
- Confirm older app builds that still fetch `comma4/install.sh` are forwarded by the shim.

### Notes

- Release bundle asset name is now `commaview-comma-<tag>.tar.gz`.
- This alpha is intentionally scoped to hook installation/platform support and release packaging cleanup, not the UDP transport rewrite.
