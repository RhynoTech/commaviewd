# Changelog

## [0.0.53-alpha] - 2026-06-05

### Improved

- Split CommaView telemetry onto a dedicated telemetry stream so video sockets can stay focused on fresh camera frames.
- Made runtime video sends bounded and freshness-first to reduce stream stalls when a client falls behind.
- Preserved local video msgq conflation behavior while adding explicit contract coverage so older frames do not pile up behind newer ones.
- Added transport v2 policy support for telemetry-only and video-only clients.

### Fixed

- Prevented telemetry-only clients from triggering video frame copy/send work.
- Fixed runtime stats writes so they no longer happen while holding the global runtime state mutex.
- Fixed telemetry stream video suppression policy handling.

### Validation

- Runtime unit/contract coverage was expanded for bounded send behavior, telemetry policy, split transport, and video msgq conflation.
- Intended companion Android app build includes dedicated telemetry receiver support on port `8203`.
