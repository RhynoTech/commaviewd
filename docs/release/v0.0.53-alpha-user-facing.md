# Changelog

## [0.0.53-alpha] - 2026-06-09

### Highlights

- Companion runtime for CommaView Android app `0.0.156-alpha`.
- Improves live video reliability by keeping runtime video delivery bounded, freshness-first, and resilient when the phone falls behind.
- Splits telemetry onto its own dedicated stream so telemetry updates no longer compete with camera video sockets.
- Reworks the V2 video transport to send camera frames as smaller TCP chunks, reducing reconnect and stutter risk from partial large-frame sends.

### Improved

- Added better runtime diagnostics and support capture, including bounded log retention, lifecycle markers, peer disconnect details, and authorized support-log collection.
- Added video health counters for backpressure, stale-frame drops, keyframe waits, chunk sends, and queue age.
- Preserved local video msgq conflation behavior while adding contract coverage so stale frames do not pile up behind newer frames.
- Added transport v2 policy support for telemetry-only and video-only clients.

### Fixed

- Prevented telemetry-only clients from triggering video frame copy/send work.
- Fixed runtime stats writes so they no longer happen while holding the global runtime state mutex.
- Fixed telemetry stream video suppression policy handling.

### Compatibility

- Compatible Android app: `0.0.156-alpha` / tag `app-v0.0.156-alpha`.
- This runtime expects the matching newer Android receiver for chunked video and the dedicated telemetry stream.
- Older Android app builds should stay on runtime `v0.0.52-alpha` or earlier.

### Testing focus

- Live-view startup and recovery.
- Video smoothness when the phone temporarily falls behind.
- Telemetry freshness while video is active.
- Runtime install, restart, stop, uninstall, support-log capture, and reconnection behavior.
